基本数据类型

java7引入了可以自由使用下划线来分隔数值
int a = 3_1_2;

整数类型

类型 字节数 占内存位数 值范围
byte 1 8 -2^7-2^7-1
short 2 16 -2^15-2^15-1
int 4 32 -2^31-2^31-1
long 8 64 -2^63-2^63-1

tips:

  1. 直接定义一个整数,默认为int类型
  2. 把在byte或short表数范围的整数赋给byte或short,系统会自动把这个整数当成byte或short处理
  3. 如果将超出int表数范围的整数当成long类型,需要在整数后加L(大小写均可) long l = 99999999L

进制表示

  • 二进制:以0b或0B开头
  • 八进制:以0开头
  • 十六进制:以0x或0X开头,10-15分别以a-f(不区分大小写)表示

数组

  1. 数组中的元素类型是唯一的
  2. 数组的初始化一旦完成,在内存中所占的空间将被固定下来,因此数组的长度不可改变
  3. 定义方式
  4. 初始化的方式
  5. foreach

    类

  • 静态成员不能直接访问非静态成员
  • 个数可变的形参只能处于形参列表的最后,并且一个方法中只能包含一个个数可变的形参
  • 创建一个对象时,系统先为该对象的所有实例变量分配内存(前提是该类已经被加载),然后初始化实例变量,顺序为:1-执行实例初始化块或声明实例变量时指定的初始值;2-执行构造器里指定的初始值

    继承

  1. 只能继承成员变量 方法 内部类,不能获得构造器和初始化块
  2. 单继承:一个类只能有一个直接父类
  3. 重写父类方法:两同两小一大
    3.1 方法名相同;形参列表相同
    3.2 子类方法返回值类型比父类的更小或相等;子类方法声明抛出的异常比父类的更小或相等
    3.3 子类方法的访问权限比父类的更大或相等
    3.4 重写的方法要一致:都为实例方法或都为类方法
    3.5 父类private修饰的方法,子类无法重写,如果子类有相同方法,算子类新建的方法
    3.6 通过super关键字,可以访问父类中隐藏的实例变量
  4. 子类中没有显示指定成员变量a的调用者时,查找a的顺序:
    4.1 该方法中是否有名为a的局部变量
    4.2 当前类中是否有名为a的成员变量
    4.3 当前类的直接父类中,是否包含名为a的成员变量,如果没有,则依次上溯所有父类,java.lang.Object也找不到时,则编译报错
  5. 程序创建一个子类对象时,会为该类中定义的实例变量分配内存,也会为它从父类继承得到的所有实例变量分配内存;如果子类定义了和父类相同名称的变量,系统依然会为父类中定义的被隐藏的变量分配内存空间
  6. 调用子类构造器初始化子类对象时,父类构造器总会在子类构造器之前执行
  7. 为保证父类的封装性,父类的设计原则
    7.1 尽量隐藏内部数据:用private修饰成员变量,不让子类直接访问
    7.2 不让子类随意访问 修改父类方法:尽量使用private修饰;必须使用public修饰又不希望子类重写方法的,可以用final修饰;希望方法可以被子类重写,但不希望被其他类访问,可以用protected修饰

多态

定义:相同类型的变量,调用同一个方法时呈现出多种不同的行为特征
引用的变量类型:1-编译时类型;2-运行时类型

  1. java允许把一个子类对象直接赋值给一个父类引用变量,无需任何类型转换,由系统自动完成
  2. 强制类型转换:(父类对象赋值给子类引用变量时)
    2.1 基本类型只能在数值类型间转换(整数型 字符型 浮点型)
    2.2 引用类型之间的转换只能在具有继承关系的两个类型之间进行
    2.3 转换时可能出现异常,可以先用instanceof进行判断是否可以成功转换

    重写equals()方法

  3. 自反性:对任意的x,x.equals(X)一定返回true
  4. 对称性:对任意的x,y,如果x.equals(y)返回true,那么y.equals(x)也返回true
  5. 传递性:x,y,z,x=y,y=z时,x=z
  6. 一致性:对任意的x,y,如果对象中用于等价比较的信息没有改变,那么无论调用equals()多少次,返回的结果应保持一直
  7. 非空性:对任何不是null的x,x.equals(null)一定返回false

    final修饰符

    成员变量:必须由程序员显式地指定初始值

    类变量,实例变量指定初始值的方式:
  8. 类变量:静态初始代码块;声明时指定(只能2选1)
  9. 实例变量:非静态初始代码块;声明时;构造器(只能3选1)

    局部变量:必须由程序员显式地指定初始值

    指定初始值方式:
  10. 定义时指定默认值
  11. 定义时未指定默认值,可以在后面赋值一次(只能一次)

    基本类型变量和引用类型变量区别

  12. 基本类型变量不能重新赋值
  13. 引用类型变量引用的地址不变(引用同一个对象),但对象可以发生改变

    宏替换

    满足下面三个条件时,final变量相当于一个直接量,编译器会把程序中所有用到该变量的地方直接替换成该变量的值:
  14. final修饰符修饰
  15. 定义变量时指定了初始值
  16. 该初始值在编译时就确定下来了
    除了可以直接为final变量赋值外,如果被赋的表达式是基本的算术表达式或字符串连接运算,没有访问普通变量,调用方法,编译器同样会把final变量当成宏变量处理

    方法:不可被重写;可以重载

    类:不能有子类

    不可变类:创建该类的实例后,实例变量不可变(java的8个包装类,Java.lang.String都是不可变类)

    自定义不可变类规则:
  17. 使用private和final修饰该类的成员变量
  18. 提供带参数的构造器(或返回该实例的类方法),用于根据传入参数初始化类里的成员变量
  19. 仅为该类的成员变量提供getter方法
  20. 如有必要,重写hashCode()和equals()方法

    抽象类

  21. 有抽象方法的类只能被定义为抽象类
  22. 必须用abstract修饰符定义类和方法,方法不能有方法体
  23. 不能被实例化,无法用new关键字来调用构造器创建抽象类的实例
  24. 可以包含成员变量,方法,构造器(主要用于被子类调用),初始化块,内部类(接口,枚举)

    接口

  25. 不能包含构造器和初始化块
  26. 可以包含静态常量的成员变量;抽象实例方法,类方法,默认方法,私有方法;内部接口,枚举
  27. 可以多继承

    内部类

  28. 提供了更好的封装
  29. 可以直接访问外部类的私有数据,但外部类不能访问内部类的实现细节
  30. 适合用于仅需要一次使用的类
  31. 可以使用修饰符:private protected static(外部类不可以使用)
  32. 非静态内部类不能拥有静态成员

    成员内部类

    非静态内部类
  33. class文件有两个Out.class Out$In.class
  34. 内部类方法访问变量的查找顺序:方法内 > 方法所在的内部类 > 内部类所在的外部类
  35. 和外部类成员变量同名,可用this 外部类名.this来区分
  36. 外部类访问非静态内部类的实例成员,必须显式创建内部类对象来调用访问实例成员
  37. 非静态内部类必须寄生在外部类对象里,而外部类对象不必一定有非静态内部类
  38. 不能有静态方法,静态成员变量,不能有静态初始化块,可以有普通初始化块
  39. 其他类中调用内
    静态内部类
  40. 寄生在外部类的类本身,而不是实例中(与外部类的类相关,而不是实例相关)
  41. 实例方法不能访问外部类的实例成员
  42. 其他类中调用
    方法内部类
  43. 不能使用访问控制符修饰
  44. 不能用static修饰
    匿名内部类
  45. 必须继承一个父类或实现一个接口(只能一个父类或一个接口)
  46. 不能是抽象类
  47. 不能定义构造器(因为没有类名),可以定义初始化块
  48. java8之前,要求被局部内部类或匿名内部类访问的局部变量必须由final修饰,java8之后:局部变量相当于自动使用了final修饰

    Lambda

    函数式接口

支持将代码块作为方法参数,用于创建只有一个抽象方法的接口

  1. 形参列表:只有一个参数时,可省略圆括号
  2. 代码块:只包含一条语句时,可省略花括号;只有一条return,可省略return关键字
  3. 目标类型必须是明确的函数式接口
  4. 只能为函数式接口创建对象
  5. @FunctionalInterface修饰的类，编译器会检测该类是否只有一个抽象方法或接口

    方法引用与构造器引用

  6. 特定对象的实例方法 特定对象::实例方法
  7. 类方法 类名::类方法
  8. 构造器 类名::new

    枚举类

  9. 继承java.lang.Enum
  10. 使用enum定义,非抽象的枚举类默认用final修饰
  11. 枚举类的构造器只能用private访问控制符
  12. 枚举类的所有实例必须在枚举类的第一行显式列出,系统会自动添加public static final
  13. 可以定义成员变量,方法和构造器
  14. 可以实现一个或多个接口
  15. 非抽象枚举类默认用final修饰;包含了抽象方法,就是抽象枚举类,弄人用abstract修饰
