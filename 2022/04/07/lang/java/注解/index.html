定义

代码中的特殊标记，这些标记可以在编译、类加载、运行时被读取，并执行相对应的处理

特点

传统的方式，我们是通过配置文件(xml文件)来告诉类是如何运行的,有了注解技术以后，我们就可以通过注解告诉类如何运行

基本Annotation

java.lang包下的5个基本注解

1. @Override

告诉编译器要检查该方法是实现父类的…可以帮我们避免一些低级的错误:

比如，我们在实现equals()方法的时候，把euqals()打错了，那么编译器就会发现该方法并不是实现父类的，与注解@Override冲突，于是就会给予错误

2. @Deprecated

Java在设计的时候，可能觉得某些方法设计得不好，为了兼容以前的程序，是不能直接把它抛弃的，于是就设置它为过时

3. @SuppressWarnings

让编译器不给予我们警告

4. @SafeVarargs

Java 7“堆污染”警告

堆污染:当把一个不是泛型的集合赋值给一个带泛型的集合的时候

5. @FunctionalInterface

用来指定该接口是函数式接口
