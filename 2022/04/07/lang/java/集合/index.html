Collection

集合与数组对比

长度

数组长度固定,集合可变

内容

数组存储同一类型的元素,集合可存储多种类型(一般不这么做)

元素的数据类型

数组可以存储基本数据类型,也可以存储引用类型;

集合只能存储引用类型(你存储的是简单的int，它会自动装箱成Integer)

Set

多线程时保护数据

HashSet

  1. 无序:排列顺序可能与添加顺序不同
  2. 线程不安全
  3. 元素值可以为null
  • 重写hashCode()的基本规则:
  1. 程序运行中,同一个对象多次调用hashCode()方法应返回相同的值
  2. 两个对象通过equals()方法比较返回true时,他们的hashCode()方法返回的值应相等
  3. 对象中用作equals()方法比较的实例变量,都应该用于计算hashCode值

    LinkedHashSet

    HashSet的子类:用链表维护元素的次序,元素的顺序与添加顺序一致

    TreeSet

  • 根据元素实际值的大小排序,而不是元素的插入顺序,是红黑树的数据结构来存储元素
  • 添加到TreeSet的对象的类,必须实现Comparable接口;
  • 向TreeSet中添加的应该是同一个类的对象,否则会引发ClassCastException
  1. 自然排序
  2. 定制排序

    EnumSet

  3. 元素必须是指定枚举类型的枚举值
  4. 有序
  5. 不允许加入null元素
  6. 内部以位向量的形式存储

    List

    ArrayList

  7. 基于数组实现
  8. 封装了一个动态,允许再分配的Object[]数组
  9. 线程不安全

    LinkedList

Vector

  1. 基于数组实现
  2. 封装了一个动态,允许再分配的Object[]数组

    Queue

  3. 先进先出
  4. 不允许随机访问队列中的元素

    ArrayDeque

    Deque为Queue的子接口,代表一个双端队列,ArrayDeque为Deque的实现类

    PriorityQueue

  5. 保存队列元素的顺序不是按加入顺序,而是队列元素的大小进行重新排序
  6. 不允许插入null元素
  7. 与TreeSet对元素的要求基本一致

    LinkedList

    实现了Deque接口,可以作为List集合,双端队列,栈使用

    Map

    HashMap

    TreeMap

    HashTable

    EnumMap
